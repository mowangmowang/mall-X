## 4. 中间件安装与配置

### 4.1 MySQL

- **版本**：5.7.x 或 8.0.x 均可（docker-compose 用 5.7，SQL dump 元信息 5.7.19）
- **端口**：3306
- **数据库**：`mall`（dev），`mall-port`（mall-portal 用的 mongo 库名，注意区分）
- **dev 账号**：`root` / `root`
- **prod 账号**：`reader` / `123456`（仅读，详见 `application-prod.yml`）
- **字符集**：`utf8mb4` / `utf8mb4_unicode_ci`（SQL 顶部 `SET NAMES utf8mb4`）
- **时区**：`Asia/Shanghai`（JDBC URL 已固定）
- **JDBC 参数**：`useUnicode=true&characterEncoding=utf-8&serverTimezone=Asia/Shanghai&useSSL=false&allowPublicKeyRetrieval=true`
- **初始化**：

  ```bash
  mysql -uroot -proot < document/sql/mall.sql
  ```

- **注意**：8.0 导入时若遇 collation 警告可忽略；`utf8_general_ci` 仍是合法 collation。

### 4.2 Redis

- **版本**：5.0+（docker 用 7，Spring Boot 3 默认 Lettuce 客户端）
- **端口**：6379
- **dev 地址（Windows + WSL2 工作流）**：`172.23.31.76:6379`（WSL2 端口转发，**非 localhost**）
- **Windows 原生 Redis**：修改 `application-dev.yml` 改为 `localhost:6379`
- **凭据**：dev 无密码；prod 由容器编排注入
- **逻辑 DB**：0
- **依赖范围**：mall-common / mall-admin / mall-portal 用了；**mall-search 主动排除**（`mall-search/pom.xml:36-41`）；**mall-ai 不用**

### 4.3 RabbitMQ

- **版本**：3.8+（docker 用 3.9.11-management）
- **端口**：5672（AMQP）/ 15672（Management UI）
- **vhost**：**`/mall`**（**非默认 `/`** — 详见 AGENTS.md:62）
- **dev 凭据**：`mall` / `mall`（在 `/mall` vhost 上授权 administrator）
- **创建 vhost + 用户**（一次性）：

  ```bash
  # 浏览器访问 http://localhost:15672 → 用 guest/guest 登录
  # Admin → Users → Add user: mall / mall
  # Admin → Virtual Hosts → Add: /mall
  # Users → mall → Set permissions: /mall 打满
  ```

- **可选插件**：`rabbitmq_delayed_message_exchange`（仅延迟消息业务需要）

### 4.4 Elasticsearch

- **版本**：**8.11.0**（docker-compose）/ **8.19.16**（AGENTS.md 推荐本地装）
- **客户端 BOM**：8.18.8（由 Spring Boot 3.5.14 管理，**禁止手动 pin**）
- **端口**：9200（HTTP）/ 9300（transport，仅容器内）
- **dev 关键配置**（`elasticsearch.yml`）：

  ```yaml
  xpack.security.enabled: false
  xpack.security.http.ssl.enabled: false
  xpack.security.transport.ssl.enabled: false
  ```

  > 8.x 默认强制 HTTPS，否则客户端连不上会报 `plaintext http traffic on an https channel`

- **必装 IK 分词器**（**`infinilabs/analysis-ik`**，从 medcl 迁移到 infinilabs）：

  ```bash
  E:\ElasticSearch\elasticsearch-8.19.16\bin\elasticsearch-plugin install -b \
    https://get.infini.cloud/elasticsearch/analysis-ik/8.19.16
  ```

- **分词策略**：
  - 索引：`analyzer = ik_max_word`（最细粒度切分）
  - 搜索：`searchAnalyzer = ik_smart`（智能切分）
  - **禁止用 smartcn**：HMM 模型，"小米 8" 会被切成 `[小, 米, 8, ...]`，破坏品牌名搜索
- **Actuator 检查**：`mall-search` 显式禁用 `management.health.elasticsearch.enabled=false`（避免假阳性）

### 4.5 MongoDB

- **版本**：4.0+（docker 用 4）
- **端口**：27017
- **dev 库名**：`mall-port`

## 6. 已知坑点（必读）

### 6.1 Maven / 构建

1. **全局跳过测试**：`<skipTests>true</skipTests>`（`pom.xml:35`）。运行单个测试：`mvn test -pl <module> -DskipTests=false -Dtest=ClassName`
2. **远程 Docker 不可达会挂**：`mvn install` 触发 `docker:build` 推送到 `192.168.3.101:2375`（`pom.xml:36, 248-255`）。公网用户必须加 `-Ddocker.skip=true`
3. **多模块级联依赖**（见 5.3）：改公共模块后必须 mvn install 两轮
4. **MyBatis mapper locations 双路径**：`classpath:dao/*.xml` + `classpath*:com/**/mapper/*.xml`
5. **mall-mbg 是生成代码**：`mall-mbg/src/main/java` 不可手改，要改 `generatorConfig.xml` 重新生成

### 6.2 Spring Boot 3 升级相关

6. **Spring Security 6 CORS 必须在 SecurityFilterChain 显式启用**：仅注册 `CorsFilter` Bean 不够；必须 `.cors(c -> c.configurationSource(...))`
7. **CORS 集中在 mall-common-cors**：
   - admin/portal 通过 `SecurityConfig.filterChain().cors(...)` 接入
   - search/ai 通过 `FilterRegistrationBean<HIGHEST_PRECEDENCE>` 注册 servlet filter
8. **`@Bean` 方法名不能与 `@Configuration` 类名同名**（否则 `BeanDefinitionOverrideException`）
9. **`setAllowCredentials(true)` 不能配 `setAllowedOrigins(["*"])`** —— 用 `addAllowedOriginPattern("*")`
10. **注入 `CorsConfigurationSource` 必须用 `@Qualifier("corsConfigurationSource")`**（`MvcHandlerMappingIntrospector` 也实现了该接口）

### 6.3 中间件

11. **Redis dev 地址是 WSL2 转发**（`172.23.31.76:6379`）—— Windows 原生 Redis 用户改 `application-dev.yml`
12. **ES 8.x 本地必须关 xpack.security**（4.4 节有完整 yml）
13. **IK 分词器来源换源**：`infinilabs/analysis-ik`（不是 `medcl/elasticsearch-analysis-ik`），下载源 `https://get.infini.cloud/elasticsearch/analysis-ik/8.19.16`
14. **禁止用 smartcn** 分词器（破坏品牌名）
15. **RabbitMQ vhost = `/mall`**，凭据 `mall/mall`
16. **MinIO API = 9000，Console = 9001**（别填反）
17. **OSS 图片 `<img>` 直连必 403**：走 `mall-common-pic` 的 `/pic/proxy?url=...`

### 6.4 安全配置

18. **JWT secret 硬编码在 `application.yml`**，每个服务一个，**必须 ≥ 64 字符**（HS512）；生产用 `openssl rand -base64 64` 替换
19. **敏感配置 gitignore**：`application-local.yml` / `application-secret.yml` / `*.env` / `.env.local`
20. **`application-prod.yml` 用了 `192.168.3.101:9090`**（MinIO），与 docker-compose 主机端口 9091 不一致，部署时核对

### 6.5 前端

21. **Vite 真实端口 3000**（`vite.config.ts:104`），**不是 AGENTS.md / README 写的 5173**
22. **Vite 强制 `host: '127.0.0.1'`**（`vite.config.ts:105`）—— 用 `localhost` 在某些 Windows 上会被解析为 `::1` (IPv6) 导致 ECONNREFUSED

### 8.3 故障排查 FAQ

**Q: `mvn install` 卡在 `docker:build`，连不上 192.168.3.101:2375？**  
A: 加 `-Ddocker.skip=true` 跳过 Docker 构建。

**Q: Redis 客户端连 `localhost:6379` 超时？**  
A: 当前是 WSL2 转发到 `172.23.31.76:6379`。改 `application-dev.yml`。

**Q: ES 启动后 Spring Boot 报 `plaintext http traffic on an https channel`？**  
A: ES 8.x 默认启用了 xpack.security.http.ssl，本地 dev 必须在 `elasticsearch.yml` 全关。

**Q: Vite 启动后浏览器报 `ECONNREFUSED ::1:3000`？**  
A: 用 `127.0.0.1:3000` 而非 `localhost:3000`（vite.config.ts 强制 IPv4）。

**Q: mall-admin 改了 `mall-security` 源码后启动还是旧的？**  
A: 多模块级联依赖暗坑。先 `mvn install -pl mall-security -am -DskipTests -Ddocker.skip=true`。

**Q: 前端 `<img src="oss-cn-shenzhen.aliyuncs.com/...">` 加载 403？**  
A: 走 `mall-common-pic` 的 `/pic/proxy?url=...`；Service 层 `ImageUrlRewriter` 自动改写。

**Q: mall-ai 启动报 401 Invalid API Key？**  
A: `application-local.yml` 写 `spring.ai.openai.api-key=sk-xxx`（gitignored）。
