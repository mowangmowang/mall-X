#### 问题2: 并发安全问题
**现状**: 
- 库存扣减未使用乐观锁
- 存在超卖风险

**影响**: 
- 论文答辩时可能被质疑
- 实际生产环境不可用

**改进建议**:
```xml
<!-- 使用乐观锁更新库存 -->
<update id="reduceStock">
    UPDATE pms_sku_stock 
    SET stock = stock - #{quantity},
        lock_stock = lock_stock + #{quantity}
    WHERE id = #{skuId} 
      AND stock >= #{quantity}
</update>
```

**工作量**: 1-2天  
**优先级**: P0

## 五、答辩准备建议

### 5.1 可能被问到的问题

#### 技术类问题
1. **为什么选择Spring Boot而不是Spring Cloud？**
   - 答：项目规模适中，单体架构更易维护；预留了微服务演进空间

2. **JWT和Session的区别？为什么选择JWT？**
   - 答：无状态、适合分布式、减轻服务器压力

3. **如何防止超卖？**
   - 答：当前使用数据库事务，建议升级为乐观锁或Redis原子操作

4. **Redis缓存穿透、雪崩如何处理？**
   - 答：布隆过滤器、空值缓存、过期时间随机化

#### 业务类问题
1. **订单超时自动取消如何实现？**
   - 答：RabbitMQ延迟队列或Redis过期监听

2. **商品搜索为什么用Elasticsearch？**
   - 答：支持全文检索、分词、模糊匹配，性能优于MySQL LIKE

3. **权限模型是如何设计的？**
   - 答：RBAC模型，用户-角色-权限三层结构

#### 创新类问题
1. **你的项目有什么创新点？**
   - 答：动态权限控制、多数据源整合、完整的电商业务流程

2. **与传统电商系统相比有什么优势？**
   - 答：前后端分离、RESTful API、模块化设计、易扩展
