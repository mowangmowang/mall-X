# 商品审核功能修复说明

## 📋 问题描述

mall-admin-web 前端项目缺少完整的商品审核功能实现：
- ✅ 后端API已存在：`/product/update/verifyStatus`
- ✅ 数据库表已存在：`pms_product_vertify_record`
- ❌ 前端缺少审核操作API调用
- ❌ 前端缺少审核操作界面

## 🔧 修复内容

### 1. 添加审核API接口

**文件**: `src/apis/product.ts`

新增API函数：
```typescript
/**
 * 批量修改审核状态
 */
export function productUpdateVerifyStatusAPI(params: { 
  ids: string; 
  verifyStatus: number; 
  detail: string 
}) {
  return http({
    url: '/product/update/verifyStatus',
    method: 'post',
    params: params,
  })
}
```

#### 2.3 添加审核处理函数

**审核操作函数**：
```typescript
// 处理商品审核
const handleVerifyProducts = async (ids: number[], verifyStatus: number) => {
  verifyOperateData.ids = ids
  verifyOperateData.verifyStatus = verifyStatus
  verifyOperateData.detail = ''
  verifyOperateDialogVisible.value = true
}
```

**确认审核函数**：
```typescript
// 确认审核操作
const handleVerifyConfirm = async () => {
  if (!verifyOperateData.detail) {
    ElMessage({
      message: '请输入审核详情',
      type: 'warning',
      duration: 1000
    })
    return
  }
  
  try {
    await productUpdateVerifyStatusAPI({
      ids: verifyOperateData.ids.join(','),
      verifyStatus: verifyOperateData.verifyStatus,
      detail: verifyOperateData.detail
    })
    ElMessage({
      message: verifyOperateData.verifyStatus === 1 ? '审核通过' : '审核不通过',
      type: 'success',
      duration: 1000
    })
    verifyOperateDialogVisible.value = false
    getList()
  } catch (error) {
    console.error('审核操作失败:', error)
    ElMessage({
      message: '审核操作失败',
      type: 'error',
      duration: 1000
    })
  }
}
```

## 🎯 功能特性

### 1. 批量审核
1. 在商品列表中勾选多个商品
2. 在批量操作下拉框中选择"审核通过"或"审核不通过"
3. 点击"确定"按钮
4. 在弹出的对话框中输入审核详情
5. 确认提交

### 2. 单个商品审核
1. 在商品列表的操作列中找到目标商品
2. 点击"通过"或"拒绝"按钮
3. 在弹出的对话框中输入审核详情
4. 确认提交

### 3. 查看审核记录
1. 在商品列表中点击"审核详情"链接
2. 查看该商品的历史审核记录，包括：
   - 审核时间
   - 审核人
   - 审核状态
   - 审核详情

## 📊 审核状态说明

| 状态值 | 说明 | 显示文本 |
|--------|------|----------|
| 0 | 未审核/审核不通过 | 未审核 |
| 1 | 审核通过 | 审核通过 |

## 📝 注意事项

1. 审核详情为必填项，不能为空
2. 审核操作会同时更新商品的 `verify_status` 字段和插入审核记录
3. 后端会自动记录审核人和审核时间
4. 审核操作不可逆，请谨慎操作
