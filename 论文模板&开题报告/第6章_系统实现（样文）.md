# 第6章 系统实现

本章介绍了系统设计完成后，系统实现的效果。本章对每一功能模块的使用进行了说明，并且对重要的关键模块的实现流程进行了阐述。

## 6.1 系统公共功能实现

### 6.1.1 系统登录模块实现

系统的登录模块界面背景是一张图片，界面正中间有一个人事管理的登录框，款中间有三个输入框，用户通过输入用户名、相应的密码和验证码相应三部分信息登录系统。用户在登录时可以选择记住密码，在下次进入界面时浏览器就可以自动填充密码。系统的验证码设定为简单两位数加减乘除计算，用户在登录时需要进行简单的数学计算，然后输入计算结果，验证码在每次重新进入系统或更新网址和用户界面时会更新。系统登录模块实现界面如图 6.1 所示。

![图 6.1 系统登录界面](imgs/img_in_image_box_294_773_980_1152.jpg)

用户登录访问后由 Spring Security 的 WebSecurityConfigurerAdapter 类实现认证和授权校验，登录成功后跳转主页，之后的每一次访问都会通过校验用户角色，角色基于 Ant 格式的 URL 权限的匹配。用户每次在登录过程中，会通过 Filter 判断用户是否登录，并且通过用户名判断用户拥有的角色，系统加载出角色列表，如果出现用户无权限访问异常，则系统进行异常处理，若用户角色权限正常则加载该用户的拥有权限的菜单。系统使用 Spring Security 授权流程图如图 6.2。

![图 6.2 Spring Security 授权流程图](imgs/img_in_image_box_366_155_865_620.jpg)

## 6.7 系统通讯模块实现

系统通讯模块为系统的交互模块，用户可以通过通讯模块和其它使用系统的用户联系，方便企业员工使用公司内部渠道相互沟通。系统通讯模块即实现了系统中点对点的在线通讯功能，此功能的实现，从前端来讲是基于 WebSocket 实现的。WebSocket 通过 HTTP 建立通信连接，之后进行连接的通信协议都依靠专用的协议。在通信过程中，通信双方可以相互发送文字信息的数据。系统通讯模块是在 HTTP 的基础上建立的，也就是通讯发起方仍然是客户端。但是一旦确定建立 WebSocket 的通信连接，不管是服务器或者客户端，双方任意一方都可以直接向对方发送报文信息。通讯模块信息建立的处理逻辑是使用键值对构建发送消息的对象。

通讯模块实现的后端是基于 Spring 框架为 WebSocket 提供的 STOMP 协议来支持的，STOMP 是一个简单的可以相互操作的协议，通常被应用于通过中间服务器在客户端之间进行异步消息的传递。

本模块实现了系统中用户的通讯，该功能被设计为在主页上方用户头像左侧，用户点击信封图标即可进入聊天界面，聊天界面打开，之前的系统界面颜色会整体暗淡为灰色，聊天界面浮于系统界面上，用户可以通过该功能模块便捷的与系统中其它有工作关系的用户进行简单的工作沟通，具体聊天界面如图 6.21 所示。
